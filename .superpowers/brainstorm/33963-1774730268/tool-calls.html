Tool Call Rendering

A — Inline Chips

Compact tags that flow inline with text. Green check for done, yellow dot for running. Tool name + first argument visible. Minimal vertical space — good for sessions with lots of tool calls.

A
Assistant
I'll fix the validation. Let me check a few files...
✓ read_file auth.ts ▶
✓ edit_file login.ts ▼
- if (email.match(/\w+/))
+ if (email.match(/^\w+@\w+\.\w+$/))
run_tests running...

B — Expandable Cards

Each tool call is a mini-card. Collapsed by default showing tool + arg. Tap to expand and see output (e.g., diffs, file content). More visual weight but much more informative.

Assistant
I'll fix the validation. Let me check a few files...
3 TOOL CALLS ▶ expand

Found the issue — the regex wasn't anchored...

C — Grouped Block

All consecutive tool calls collapse into a single block with a count header. Expandable to see individual tools. Most compact — keeps focus on the assistant's text narrative.
